JDBC Data Store Properties

The JDBC Data Store properties panel is used by the Mirror Data Stores screen (as part of a List Maintenance Panel), by the Edit JDBC Data Stores screen (as part of a OK-Cancel Panel) and by the Data Store Wizard.

For information on mirrors see the Mirror section below.

The JDBC Data Store properties panel is used to define the JDBC connection information used by MDN to connect to one of your company's databases.


Below is the JDBC Data Store properties panel as shown in an OK-Cancel Panel by pressing the Properties button with a Data Store selected in the Data Stores screen. A similar panel is displayed in the Data Store Wizard for entering the Data Store details.

The Name field is the name of the data store, it should describe the content or purpose of the data store and must be unique within this installation of MDN.

Description is any additional text to describe the data store.

The Driver combo box displays the set of JDBC drivers available to MDN, as configured in the JDBC Drivers screen. Select the appropriate driver for your database. You may need to install a driver for your database if a suitable driver is not shown here.

The URL field specifies the JDBC URL to connect to your database. The format of this URL is specific to the selected JDBC driver. See the documentation for the JDBC driver to determine the correct format.
The format for the JDBC/ODBC Bridge driver is jdbc:odbc:dsn, where dsn is the ODBC data source name.

Catalog is the database catalog. Usually left blank.

User is the username used to log into the database.

Password is the password used to log into the database.

The Mirrored Data Store? checkbox is used to select this Data Store for mirroring. This means that MDN will use a mirror copy of the source Data Store rather than accessing it directly. You may use Data Transfers to copy data from the source Data Store to the mirror. If mirroring is selected then a combo box will be displayed to allow the selection of the desired Mirror Data Store.


Mirror Data Stores

A Mirror Data Store is a location where mirrored data is stored separately from the source data. This is distinct from a Mirrored Data Store which defines the source of the data, but has the Mirrored Data Store? box checked to indicate that MDN will access the mirror copy rather than the source.

To use mirroring you must first define one or more Mirror Data Stores, using the Mirror Data Stores screen shown below, and then select a Data Store and enable mirroring for it by checking the Mirrored Data Store? checkbox for that Data Store using its properties screen.

This screen has the same fields as the Edit JDBC Data Stores screen with the exception of the Mirrored Data Store? checkbox, as it is not valid for a Mirror Data Store to be mirrored.

The Mirror Data Stores screen uses a List Maintenance Panel to show the list of mirrors, add mirrors, delete mirrors and edit the attributes of a mirror.
