JDBC Drivers


The JDBC drivers screen is a List Maintenance screen that you use to maintain the set of JDBC drivers recognised by Mobile Data Now.

Before you use this screen to add a new JDBC driver for use by MDN you must install the JAR file for the driver into the lib directory under the MDN install directory. Then you must edit the mdnserver.lax file to add the location of the JDBC driver to the lax.class.path. The lax.class.path sets the CLASSPATH used by MDN Server to locate its JARS.

For example if your new JDBC driver is called driver.jar and you installed MDN to c:\mdn then driver.jar should be placed in c:\mdn\lib and the text ;c:\mdn\lib\driver.jar should be appended to the end of the line in mdnserver.lax that begins with lax.class.path=.

See the Installation and Configuration guide for more information on starting MDN, the MDN startup scripts and setting classpaths.

Press the Clear button, then enter the driver information and press Insert.

The Name field is the name you want to use to refer to the JDBC driver.

The Driver field is the fully qualified package and class name of the JDBC driver. The driver documentation will indicate what name you should place in this field.

Description is any addition text describing the driver.
