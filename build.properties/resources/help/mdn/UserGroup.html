Users and Groups

A User represents a person who may log in to MDN either as an administrator or as a WAP user. Users belong to Groups that define their privileges and, in the case of WAP users, what menus, views and fields they can access.

To create users, assign them to groups and set privileges for those groups see the Users screen and the Groups screen.

You can switch between the Users, Groups and Data View Permissions screens by clicking on the desired tab.

Note that group privileges are not currently required as the MDN Administrator does not restrict function based on privileges.

The presentation of menus to WAP users is specified at the Group level, so when creating users and assigning them to groups you will need to create separate groups for users that need different WAP menus.


Data View Permissions


The Data View Permissions screen is used to limit which views and view fields are accessible to a given group. In general what a WAP user can see is determined by the structure of their group's menus in the Presentation Builder and by the fields included in the definition of the views. For security reasons it may be desirable for the members of a given group to see a restricted set of fields.

To restrict the view or field permissions for a group expand that group and view in the tree, select a view or field and press the Enable or Disable button to grant or restrict view access to the field or view.

Note that if a view field has been designated as a Naming Field then that field will still be shown as a part of the record name even if the field permission has been removed. The disabled field will not be shown in view or edit screens.

If a View is selected pressing the Properties button will display the Group Data View screen.


Group Data View


The Group Data View screen allows you to control what changes members of a group may make to the selected view. The Allow Add, Allow Edit and Allow Delete check boxes control whether the user may add, edit or delete records in the view.
